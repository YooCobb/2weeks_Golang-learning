Go 언어의 큰 특징
1. 정적 타입, 강 타입
	컴파일시 변수의 자료형이 변하는 것은 동적인 것임. 파이썬의 경우 코드상 자료형이 없으며 c/c++에서는 자료형의 선언이 있음
	
	약 타입은 컴파일 또는 실행 시점에 정해진 규칙에 따라 암묵적으로 형변환이 일어나는 것을 말함
	int a = 1;
	float b = 1.3f;

	float c = a + b; // c/c++ 에서도 int형인 a가 float형으로 형변환이 일어남

3. 가비지 컬렉션(GC)
	c/c++은 메모리를 할당하면 반드시 해제해주어야함. 로직작성보다 메모리 관리에 더 많은 노력과 시간을 소모하게 되어 생산성이 떨어지게됨
4. 병행성★★★★★
	이 책을 펴기 전부터 go에 대해 검색하면 반드시 나오던 고루틴!!!
	

	언어의 차원에서 병행성을 제공함.
	병행성 : 단일 코어에서 스레드를 여러 개가 시간을 쪼개어 순차적으로 실행하는 것. 논리적 개념이기 때문에 단일 코어, 멀티 코어는 차이 없다.
	병렬성 : 동시에 처리하는 물리적인 개념으로 작업을 여러 cpu 코어에 나워서 동시에 처리하느 상태

	go는 go키워드를 통해 함수 여러 개를 동시에 실행할 수 있음.
	스레드와 차이점은 운영체제의 커널에서 제공하는 리소스이기 때문에 많이 생성할수록 부담이 커짐.
	고는 적정량의 스레드를 생성해서 고루틴을 처리함.
	코어 개수  설정에 따라 멀티코어도 지원함.

	다양한 네트워크 아리브러리를 제공하므로 인터넷 프로그래밍에 유용함.
